FIRST STEPS (DO THIS NOW)
================================================================================

Step 1: Update Configuration
-----------------------------
Edit these files with your values:

1. backend-setup.tf (line 20)
   bucket = "YOUR-UNIQUE-BUCKET-NAME"

2. main.tf (line 13)
   bucket = "YOUR-UNIQUE-BUCKET-NAME"  (same as above)

3. terraform.tfvars (line 6)
   key_name = "your-aws-key-pair"  (or leave empty "")

Step 2: Configure AWS Credentials
----------------------------------
Run: aws configure

Or set environment variables:
  AWS_ACCESS_KEY_ID
  AWS_SECRET_ACCESS_KEY

Step 3: Deploy Infrastructure
------------------------------
Run these commands:
  terraform init
  terraform apply

Step 4: Migrate to Terraform Cloud
-----------------------------------
Follow: GET_STARTED.md (Step 3)

================================================================================
WHAT THIS PROJECT CREATES
================================================================================

AWS Resources:
--------------
- 1 VPC (10.0.0.0/16)
- 2 Public Subnets (across 2 AZs)
- 1 Internet Gateway
- 1 Route Table
- 2 EC2 Instances (t2.micro with Nginx)
- 1 Security Group
- 1 S3 Bucket (for state)
- 1 DynamoDB Table (for locking)

Estimated Cost: ~$20-25/month
(Remember to destroy when done testing!)

================================================================================
IMPORTANT NOTES
================================================================================

Security:
---------
- NEVER commit .tfvars files with secrets to Git
- Always mark sensitive variables in Terraform Cloud
- Use .gitignore (already configured)
- Review plans before applying

Best Practices:
---------------
- Always review terraform plan output
- Use workspaces for different environments
- Enable notifications for important events
- Tag all resources for organization
- Backup state before major changes

Cost Management:
----------------
- This project costs ~$20-25/month if left running
- Destroy resources when done testing: terraform destroy
- Use t2.micro instances for testing (free tier eligible)
- Monitor AWS billing dashboard
